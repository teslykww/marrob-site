# MARROB — лендинг для Tilda

Готовые HTML/CSS/JS блоки под **тот же контент и порядок**, что и лендинг в `kimi2/app` (React).

## Сборка экспорта

```bash
cd marrob-tilda
npm run build
```

В папке **`tilda-export/`** появятся файлы для вставки в Zero Block, код для **`<head>`** — **`00-global.html`** и **`00-вставьте-в-head.html`**, **`asset-base.config.json`** (пример настройки URL картинок), **`все-блоки-одним-файлом.html`**, **`полная-страница.html`** / **`index.html`**.  

**Картинки как на Vercel:** в **`asset-base.config.json`** задайте **`publicBaseUrl`** (URL деплоя без слэша в конце). После **`npm run build`** все изображения в экспорте получат абсолютные ссылки на ваш хостинг (те же пути, что у `kimi2/app/public/`). Пустой `publicBaseUrl` — относительные пути для локального просмотра. Подробнее — в **`tilda-export/README.md`**.

**Сейчас 21 блок Zero** на главной (как порядок секций в `kimi2/app` на Vercel: Hero с трёхстрочным h1, **8 карточек преимуществ**, Решение, каталог + лид-магнит, … контакты `#contact` перед FAQ). Отдельный mid-page блок «Получить расчёт» (`11-cta-form.html`) в сборку **не входит** — на сайте такого блока нет; файл оставлен в репозитории для ручного использования.

## Исходники блоков (корень `marrob-tilda/`)

Редактируйте `01-header.html`, `02-hero.html`, … затем снова **`npm run build`**. В `assets/` должны быть `hero-bg.png`, `solution-scheme.png`, `logo.png`, папка `certs/` — их копирует скрипт в `tilda-export/assets/`.

## Страница дилеров

- **dealers.html** — готовая отдельная HTML-страница для B2B (партнёрская программа, FOMO-строка, тёмный блок формы, «Путь партнёра», POST заявки). После правок выполняйте **`npm run build`**, чтобы обновить **`tilda-export/dealers.html`**.
- **Отправка в Bitrix24:** форма шлёт JSON на **marrob-api** (`POST /leads/dealer`), тот же контракт, что и React. Укажите URL одним из способов:
  - атрибут **`data-endpoint="https://ваш-домен/leads/dealer"`** на `<form id="marrob-dealers-form">`, или
  - глобально: **`window.MARROB_LEAD_DEALER_URL = 'https://…/leads/dealer'`** в коде страницы Tilda до скрипта формы.
- Подробности развёртывания API и чеклист проверки: **[../marrob-api/README.md](../marrob-api/README.md)**.
- Фото менеджера: положите **`images/dealer-manager.webp`** в экспорт (копируется из `kimi2/app/public/images` при сборке); иначе подставится **`images/contact-office.png`**.

## Настройки в Tilda

- **Формы:** в блоке 11 (форма) и на странице дилеров замените `action` формы на отправку в Tilda Forms или на ваш endpoint (например, форма Tilda с полями Имя, Телефон, Город, Площадь, Тип заявки).
- **Изображения:** в блоках Hero, Gallery, Trust и т.д. используются placeholder-URL (Unsplash). Замените на ваши изображения через медиабиблиотеку Tilda или загрузку на хостинг.

## Просмотр варианта Kimi (референс)

Чтобы сравнить с оригинальным дизайном Kimi:

1. Запустите вариант Kimi: в папке `kimi/app` выполните  
   `npx serve dist -p 5174`  
   (или из корня проекта: `cd kimi/app && npx serve dist -p 5174`).
2. Откройте в браузере **kimi-preview.html** (через ваш локальный сервер, например `npx serve . -p 3333` в папке marrob-tilda) или перейдите по ссылке «Референс (Kimi)» в подвале главной страницы.

Страница **kimi-preview.html** показывает вариант Kimi во фрейме (http://localhost:5174). Если сервер Kimi не запущен, на странице будет инструкция и ссылка для открытия в новой вкладке.
